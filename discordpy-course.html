Your First Discord Bot

Set up and run your first Discord.py bot.

üöÄ What is Discord.py?

Discord.py is a Python library that lets you create Discord bots. It handles all the communication with Discord's API, so you can focus on building features!

üìã Prerequisites
  • Python 3.8 or higher installed
  • A Discord account
  • A Discord server where you're admin (for testing)
  • Basic Python knowledge (see Python course)

üì¶ Installation

Terminal Shell 
# Install discord.py
pip install discord.py

# Install with voice support (optional)
pip install discord.py[voice]

# Also install python-dotenv for environment variables
pip install python-dotenv

üîë Creating Your Bot on Discord

  1. Go to Discord Developer Portal
  2. Click "New Application" and give it a name
  3. Go to the "Bot" tab and click "Add Bot"
  4. Click "Reset Token" and copy your bot token
  5. Enable "Message Content Intent" under Privileged Gateway Intents
  6. Go to OAuth2 ‚Üí URL Generator, select "bot" and "applications.commands"
  7. Select permissions and use the generated URL to invite your bot
‚ö†Ô∏è Keep Your Token Secret!

Never share your bot token or commit it to GitHub. Anyone with your token can control your bot!

üìù Your First Bot Code

.env ENV 
DISCORD_TOKEN=your_bot_token_here
bot.py Python 
import discord
from discord.ext import commands
import os
from dotenv import load_dotenv

# Load environment variables
load_dotenv()

# Set up intents (permissions for your bot)
intents = discord.Intents.default()
intents.message_content = True  # Required to read message content

# Create the bot
bot = commands.Bot(command_prefix='!', intents=intents)

@bot.event
async def on_ready():
    """Called when the bot is ready and connected."""
    print(f'Logged in as {bot.user.name} (ID: {bot.user.id})')
    print('------')

@bot.command()
async def ping(ctx):
    """Simple ping command to test the bot."""
    await ctx.send('Pong!')

# Run the bot
bot.run(os.getenv('DISCORD_TOKEN'))

‚ñ∂Ô∏è Running Your Bot

Terminal Shell 
python bot.py

If successful, you'll see "Logged in as YourBotName" in the console. Go to your Discord server and type !ping to test!

Bot Events

Learn how your bot responds to Discord events.

üì° What are Events?

Events are things that happen on Discord - messages sent, members joining, reactions added, etc. Your bot can listen for these events and respond to them.

Event Flow
Discord
‚Üí
Event
‚Üí
Your Bot
‚Üí
Response

üéØ Common Events

Event When it fires
on_ready Bot connects and is ready
on_message A message is sent in any channel
on_member_join A new member joins a server
on_member_remove A member leaves a server
on_reaction_add Someone adds a reaction

üìù Event Examples

events.py Python 
@bot.event
async def on_ready():
    """Called when bot is ready."""
    print(f'{bot.user} is online!')
    # Set bot status
    await bot.change_presence(
        activity=discord.Game(name="!help for commands")
    )

@bot.event
async def on_message(message):
    """Called when any message is sent."""
    # Ignore messages from the bot itself
    if message.author == bot.user:
        return

    # Respond to specific words
    if 'hello' in message.content.lower():
        await message.channel.send(f'Hello {message.author.name}!')

    # IMPORTANT: Process commands after custom on_message
    await bot.process_commands(message)

@bot.event
async def on_member_join(member):
    """Called when a member joins the server."""
    # Find a welcome channel
    channel = discord.utils.get(member.guild.text_channels, name='welcome')
    if channel:
        await channel.send(f'Welcome to the server, {member.mention}!')
‚ö†Ô∏è Important: process_commands()

If you use on_message, you MUST call await bot.process_commands(message) at the end, or your commands won't work!

Text Commands

Create commands that users can trigger with a prefix.

‚å®Ô∏è Prefix Commands

Prefix commands start with a symbol like ! or ?. While slash commands are now preferred, prefix commands are still useful for quick testing and admin utilities.

commands.py Python 
# Basic command
@bot.command()
async def hello(ctx):
    """Says hello to the user."""
    await ctx.send(f'Hello, {ctx.author.name}!')

# Command with arguments
@bot.command()
async def say(ctx, *, message):
    """Repeats what you say."""
    await ctx.send(message)

# Command with multiple arguments
@bot.command()
async def add(ctx, num1: int, num2: int):
    """Adds two numbers."""
    result = num1 + num2
    await ctx.send(f'{num1} + {num2} = {result}')

üìã The Context Object (ctx)

Every command receives a ctx (context) object with useful information:

context.py Python 
@bot.command()
async def info(ctx):
    # Who sent the command
    author = ctx.author           # User object
    author_name = ctx.author.name # Username
    author_id = ctx.author.id     # User ID

    # Where the command was sent
    channel = ctx.channel         # Channel object
    guild = ctx.guild             # Server object

    # The message that triggered the command
    message = ctx.message

    # Responding
    await ctx.send("Sends a message")
    await ctx.reply("Replies to the command message")

Lesson 4

Slash Commands Basics

Create modern slash commands with autocompletion.

‚ö° What are Slash Commands?

Slash commands start with / and show up in Discord's command picker with descriptions and autocomplete. They're the modern way to build bot commands!

‚úÖ Why Use Slash Commands?
  • Users can discover commands by typing /
  • Built-in argument validation
  • Descriptions and help text
  • Discord's official recommended approach

üìù Creating Slash Commands

slash_commands.py Python 
import discord
from discord import app_commands
from discord.ext import commands

# Create bot with command tree
bot = commands.Bot(command_prefix='!', intents=discord.Intents.default())

# Simple slash command
@bot.tree.command(name="ping", description="Check bot latency")
async def ping(interaction: discord.Interaction):
    latency = round(bot.latency * 1000)
    await interaction.response.send_message(f'Pong! {latency}ms')

# Slash command with argument
@bot.tree.command(name="greet", description="Greet a user")
async def greet(interaction: discord.Interaction, user: discord.Member):
    await interaction.response.send_message(f'Hello, {user.mention}!')

# Sync commands when bot is ready
@bot.event
async def on_ready():
    await bot.tree.sync()  # Sync to all servers
    print(f'Synced commands!')

üîÑ Interaction vs Context

Slash commands use interaction instead of ctx:

Prefix Command (ctx) Slash Command (interaction)
ctx.send() interaction.response.send_message()
ctx.author interaction.user
ctx.guild interaction.guild
ctx.channel interaction.channel

üì® Response Types

responses.py Python 
@bot.tree.command(name="example")
async def example(interaction: discord.Interaction):
    # Normal response
    await interaction.response.send_message("Hello!")

    # Ephemeral (only visible to user)
    await interaction.response.send_message("Secret!", ephemeral=True)

    # Deferred response (for slow operations)
    await interaction.response.defer()
    # ... do slow operation ...
    await interaction.followup.send("Done!")
‚ö†Ô∏è Response Timeout

You must respond to a slash command within 3 seconds. Use defer() if you need more time, then followup.send() when done.

üîÆ Autocomplete

autocomplete.py Python 
# Dynamic autocomplete
async def team_autocomplete(
    interaction: discord.Interaction,
    current: str
) -> list[app_commands.Choice[str]]:
    teams = ["Patriots", "Chiefs", "Eagles", "Cowboys", "49ers"]
    return [
        app_commands.Choice(name=team, value=team)
        for team in teams
        if current.lower() in team.lower()
    ][:25]  # Max 25 choices

@bot.tree.command(name="team")
@app_commands.autocomplete(team=team_autocomplete)
async def team(interaction: discord.Interaction, team: str):
    await interaction.response.send_message(f'Team: {team}')

Cogs & Extensions

Organize your bot into modular components.

üß© What are Cogs?

Cogs are classes that group related commands and events together. They make large bots easier to manage!

cogs/moderation.py Python 
from discord.ext import commands

class Moderation(commands.Cog):
    def __init__(self, bot):
        self.bot = bot

    @commands.command()
    @commands.has_permissions(kick_members=True)
    async def kick(self, ctx, member: discord.Member, *, reason=None):
        await member.kick(reason=reason)
        await ctx.send(f'{member} was kicked.')

async def setup(bot):
    await bot.add_cog(Moderation(bot))
bot.py Python 
# Load cogs
@bot.event
async def on_ready():
    await bot.load_extension('cogs.moderation')
    print('Cog loaded!')

Error Handling

Handle errors gracefully in your bot.

üõ°Ô∏è Global Error Handler

errors.py Python 
@bot.event
async def on_command_error(ctx, error):
    if isinstance(error, commands.MissingPermissions):
        await ctx.send("You don't have permission to do that!")
    elif isinstance(error, commands.MissingRequiredArgument):
        await ctx.send(f"Missing argument: {error.param.name}")
    elif isinstance(error, commands.CommandNotFound):
        pass  # Ignore unknown commands
    else:
        print(f"Error: {error}")

# Slash command error handler
@bot.tree.error
async def on_app_command_error(interaction: discord.Interaction, error):
    await interaction.response.send_message(
        f"An error occurred: {error}",
        ephemeral=True
    )
