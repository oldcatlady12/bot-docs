March Madness Bot

Overview

The March Madness Bot is a comprehensive NCAA Tournament bracket management and betting system. Create brackets, place bets on games, earn coins, and compete on leaderboards during March Madness.

Commands

The bot has 10 slash commands: 3 setup commands, 3 user commands, and 4 admin commands.

Setup Commands Admin Only

CommandDescription
/setup_channels Set up the bot channels. Requires submission_channel (for bracket submissions) and alerts_channel (for March Madness alerts).
/create_submission_button Create the bracket submission button. Posts an interactive message with a button users can click to submit their bracket.
/create_coin_collector Create coin collection buttons. Posts buttons for daily, weekly, and monthly coin claims.

User Commands Everyone

CommandDescription
/balance [member] Check your coin balance or another member's balance. Shows current coins available.
/my_bracket View your March Madness bracket and stats. Displays your bracket as an image along with stats (correct picks, accuracy percentage, elimination status, current streak).
/leaderboard <category> View March Madness leaderboards. Categories: elimination (Still Alive), accuracy (Most Correct Picks), coins (Most Coins), gambled (Most Coins Gambled), highest_bet (Highest Single Bet).

Admin Commands Admin Only

CommandDescription
/lock_submissions Lock bracket submissions. Use when the tournament starts to prevent new brackets.
/unlock_submissions Unlock bracket submissions. Re-enables bracket submissions if needed.
/update_game_result Update a game result. Requires round_number (1-6), game_number (game in that round), and winner (team name). Automatically checks elimination status.
/end_tournament End the tournament and show final results. Generates final report showing elimination winner, top predictors, and participation statistics.

Interactive UI Components

The bot uses Discord buttons and modals for interactive features:

Bracket Submission Button

Created via /create_submission_button. Users click to open a modal where they fill in their tournament bracket picks.

Coin Collection Buttons

Created via /create_coin_collector. Three buttons for:

  • Daily: Claim every 24 hours
  • Weekly: Claim every 7 days
  • Monthly: Claim every 30 days

Bracket Betting View

Buttons to place bets on brackets and view your existing bets.

Core Features

🏀 Bracket Management

Users create tournament brackets with predictions for all rounds. The advanced bracket builder supports selecting winners from First Four through the Championship game. Brackets generate visual images for easy viewing.

💰 Betting System

Place bets on brackets using coins. Track your betting history, view active bets, and see potential payouts.

💸 Economy System

Users earn coins through:

  • Daily/weekly/monthly claims
  • Correct bracket picks
  • Winning bets
  • Bonus activities

🚫 Elimination Pool

Track who's still alive based on their bracket picks. When a user's championship pick is eliminated, they're marked as eliminated from the pool.

📈 Bracket Visualization

Generate visual bracket images showing all picks, results, and accuracy statistics. Track progress through each tournament round.

Database Schema

The bot uses SQLite with these main tables:

-- Brackets table brackets: bracket_id, user_id, guild_id, bracket_data, elimination_picks, score, created_at -- Bets table bets: bet_id, user_id, guild_id, game_id, bet_type, amount, odds, status -- Economy table economy: user_id, guild_id, balance, total_earned, total_wagered, last_daily -- Game results (tournament games) game_results: game_id, round, winner, score, settled

Seasonal Usage

This bot is primarily active during the NCAA Tournament (March-April). The workflow is:

  1. Before tournament: Set up channels, create buttons, users submit brackets
  2. Tournament starts: Lock submissions with /lock_submissions
  3. Each round: Update game results with /update_game_result
  4. End: Run /end_tournament for final leaderboards
  5. Off-season: Use manage.py to reset for next year
