Learn the three ways to declare variables in JavaScript.

üì¶ Three Ways to Declare Variables

Unlike Python which just uses name = value, JavaScript has three keywords for creating variables:

Keyword Can Reassign? Scope When to Use
const No ‚ùå Block Values that won't change

‚úÖ Best Practice

Use const by default. Only use let when you need to reassign the variable. Never use var.

Conditionals

Learn if/else statements and comparison operators.

üîÄ If/Else Statements

Conditionals let your code make decisions. The syntax uses curly braces instead of Python's colons and indentation:

Python
if score >= 90:
    print("A")
elif score >= 80:
    print("B")
else:
    print("C")
JavaScript
if (score >= 90) {
    console.log("A");
} else if (score >= 80) {
    console.log("B");
} else {
    console.log("C");
}
üí° Key Differences from Python
  • Condition must be in parentheses ()
  • Use curly braces {} instead of indentation
  • else if instead of elif

‚öñÔ∏è Comparison Operators

comparisons.js JavaScript 
// Equal (loose - converts types)
5 == "5"     // true (string converted to number)
5 == 5       // true

// Strict equal (recommended - no conversion)
5 === "5"    // false (different types)
5 === 5      // true

// Not equal
5 != "5"     // false (loose)
5 !== "5"    // true (strict)

// Other comparisons
10 > 5       // true
10 < 5       // false
10 >= 10     // true
10 <= 5      // false
‚ö†Ô∏è Always Use === and !==

Always use strict equality (===) to avoid unexpected type conversion bugs. == can cause confusing results!

üîó Logical Operators

logical.js JavaScript 
// AND - both must be true
if (age >= 18 && hasLicense) {
    console.log("Can drive");
}

// OR - at least one must be true
if (isAdmin || isModerator) {
    console.log("Has permissions");
}

// NOT - inverts boolean
if (!isBanned) {
    console.log("User allowed");
}

// Python vs JavaScript
// Python: and, or, not
// JavaScript: &&, ||, !

Loops

Learn for, while, and for...of loops to repeat code.

üîÅ The For Loop

The classic for loop has three parts: initialization, condition, and increment:

forloop.js JavaScript 
// for (start; condition; step)
for (let i = 0; i < 5; i++) {
    console.log(i);  // 0, 1, 2, 3, 4
}

// Count backwards
for (let i = 5; i > 0; i--) {
    console.log(i);  // 5, 4, 3, 2, 1
}

// Skip by 2
for (let i = 0; i < 10; i += 2) {
    console.log(i);  // 0, 2, 4, 6, 8
}
Python
for i in range(5):
    print(i)
JavaScript
for (let i = 0; i < 5; i++) {
    console.log(i);
}

üîÑ For...of Loop (Iterate Arrays)

The modern way to loop through arrays (like Python's for-in):

forof.js JavaScript 
const fruits = ["apple", "banana", "cherry"];

// for...of - get each item
for (const fruit of fruits) {
    console.log(fruit);
}
// apple, banana, cherry

// If you need the index too, use entries()
for (const [index, fruit] of fruits.entries()) {
    console.log(`${index}: ${fruit}`);
}
// 0: apple, 1: banana, 2: cherry

‚ö° While Loop

while.js JavaScript 
// While - runs while condition is true
let count = 0;
while (count < 5) {
    console.log(count);
    count++;  // Don't forget this or infinite loop!
}

// Do...while - runs at least once
let num = 10;
do {
    console.log(num);  // Runs once even though condition is false
} while (num < 5);

üõë Break and Continue

control.js JavaScript 
// break - exit the loop entirely
for (let i = 0; i < 10; i++) {
    if (i === 5) break;
    console.log(i);  // 0, 1, 2, 3, 4
}

// continue - skip to next iteration
for (let i = 0; i < 5; i++) {
    if (i === 2) continue;
    console.log(i);  // 0, 1, 3, 4 (skips 2)
}

‚úèÔ∏è Modifying Arrays

modify.js JavaScript 
const arr = ["a", "b", "c"];

// Add to end
arr.push("d");        // ["a", "b", "c", "d"]

// Add to beginning
arr.unshift("z");     // ["z", "a", "b", "c", "d"]

// Remove from end

// Change an element

üîç Array Methods

methods.js JavaScript 
const nums = [1, 2, 3, 4, 5];

// find - get first matching element
nums.find(n => n > 3);       // 4

// filter - get all matching elements
nums.filter(n => n > 2);     // [3, 4, 5]

// map - transform each element
nums.map(n => n * 2);        // [2, 4, 6, 8, 10]

// includes - check if contains
nums.includes(3);           // true

// indexOf - find position
nums.indexOf(3);            // 2

// join - convert to string
nums.join(", ");            // "1, 2, 3, 4, 5"

// reduce - accumulate values
nums.reduce((sum, n) => sum + n, 0);  // 15

Arrow Functions

Master the modern, concise function syntax.

‚û°Ô∏è Arrow Function Syntax

Arrow functions are a shorter way to write functions:

arrow.js JavaScript 
// Traditional function
function add(a, b) {
    return a + b;
}

// Arrow function (full syntax)
const add = (a, b) => {
    return a + b;
};

// Arrow function (implicit return - one line)
const add = (a, b) => a + b;

// Single parameter - no parentheses needed
const double = x => x * 2;

// No parameters - empty parentheses required
const greet = () => "Hello!";

üìã Arrow Functions with Arrays

Arrow functions shine with array methods:

arrays.js JavaScript 
const numbers = [1, 2, 3, 4, 5];

// map - transform each element
const doubled = numbers.map(n => n * 2);
// [2, 4, 6, 8, 10]

// filter - keep matching elements
const evens = numbers.filter(n => n % 2 === 0);
// [2, 4]

// find - get first match
const firstBig = numbers.find(n => n > 3);
// 4

// sort - custom sorting
const sorted = numbers.sort((a, b) => b - a);
// [5, 4, 3, 2, 1] (descending)

// Chaining methods
const result = numbers
    .filter(n => n > 2)
    .map(n => n * 10);
// [30, 40, 50]

üîô Returning Objects

Wrap object literals in parentheses:

objects.js JavaScript 
// Wrong - JS thinks {} is a code block
const makeUser = name => { name: name };  // undefined!

// Correct - wrap in parentheses
const makeUser = name => ({ name: name });
// Or with shorthand
const makeUser = name => ({ name });

console.log(makeUser("Arthur"));
// { name: "Arthur" }

Promises & Async/Await

Master asynchronous JavaScript - essential for Discord bots!

‚è≥ What is Asynchronous Code?

Async code lets your program continue while waiting for slow operations (API calls, database queries, etc.):

Synchronous vs Asynchronous
Task 1
‚Üí
Wait...
‚Üí
Task 2

Sync: Must wait. Async: Can continue while waiting!

ü§ù Promises

A Promise represents a value that will be available later:

promises.js JavaScript 
// A Promise has 3 states: pending, fulfilled, rejected

// Using .then() and .catch()
fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.log('Error:', error);
    });

// Code here runs immediately (doesn't wait!)
console.log('This prints first!');

‚ú® Async/Await (Modern Way)

Makes async code look synchronous and easier to read:

asyncawait.js JavaScript 
// async function can use await
async function getData() {
    try {
        const response = await fetch('https://api.example.com/data');
        const data = await response.json();
        console.log(data);
        return data;
    } catch (error) {
        console.log('Error:', error);
    }
}

// Arrow function version
const getData = async () => {
    const response = await fetch('https://api.example.com/data');
    return response.json();
};
üí° Key Rules
  • await can only be used inside an async function
  • await pauses execution until the Promise resolves
  • Always use try/catch for error handling
  • Async functions always return a Promise
Python (asyncio)
async def get_data():
    data = await fetch(url)
    return data

// CommonJS exports
function formatNumber(num) {
    return num.toLocaleString();
}

function randomInt(min, max) {
    return Math.floor(Math.random() * (max - min + 1)) + min;
}

module.exports = { formatNumber, randomInt };
