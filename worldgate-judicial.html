Overview

The WorldGate Judicial Bot is a comprehensive judicial system for Discord servers. It allows server administrators to manage reports, cases, and appeals in an organized manner with voting functionality and multi-server broadcasting.

📝

Case Filing

Report users, servers, or websites with evidence

⚖

Appeals System

Convicted users can appeal verdicts via Discord or web

🗳

Jury Voting

Dungeon Jury members can vote on cases

🌐

Web Interface

Submit reports and appeals via website

📡

Multi-Server

Broadcast cases across multiple Discord servers

🔒

Case Records

Permanent records of all guilty verdicts

Setup & Installation

Prerequisites

  • Node.js 18.x or higher
  • Discord Bot Token
  • A Discord server with appropriate permissions

Installation Steps

# Navigate to the bot directory cd worldgate-judicial-bot # Install dependencies npm install # Copy example environment file cp .env.example .env # Edit .env with your bot token # Edit config.json with your server IDs # Deploy slash commands node deploy-commands.js # Start the bot node index.js

Required Files

File Purpose
.env Contains DISCORD_TOKEN and other secrets
config.json Bot configuration (server IDs, owner ID, etc.)
worldgate.db SQLite database (auto-created)

Configuration

config.json Structure

{ "clientId": "YOUR_BOT_CLIENT_ID", "mainServerId": "YOUR_MAIN_SERVER_ID", "botOwnerId": "YOUR_DISCORD_USER_ID", "discordInvite": "https://discord.gg/your-invite", "contactChannelId": "CONTACT_CHANNEL_ID" }
ⓘ
Important: The mainServerId should be the primary WorldGate server where cases are filed and the court-house forum exists.

Case Management

Filing a Report

Users can file reports against:

  • Users - Discord user ID required
  • Servers - Server ID or invite link
  • Websites - Full URL of malicious site

Case Workflow

1. Report Filed

User submits report via /report command or web form. Creates a forum thread in #court-house.

2. Evidence Upload

Reporter can upload evidence files (images, documents) to the case thread.

3. Jury Voting

Dungeon Jury members vote Guilty or Not Guilty on the case.

4. Verdict

Bot owner closes case with final verdict. Forum thread is deleted, case moved to #case-logs if guilty.

Case Numbers

Cases are automatically assigned sequential numbers in the format: CASE-000001

Appeals System

Appeal Methods

Convicted users can appeal through three methods:

💻

Website Form

Submit appeal via the Appeals tab on the web interface

👥

Server Join

When convicted user joins, appeal dropdown appears in #appeals

🔘

Case-Logs Button

Click "Appeal This Verdict" button on guilty case posts

Voting System

Who Can Vote

  • Users with the "Dungeon Jury" role
  • Bot owner (as configured in config.json)

Vote Types

🔴 Vote Guilty

Defendant is guilty of the offense

🟢 Vote Not Guilty

Defendant is not guilty or insufficient evidence

Closing Cases

Only the bot owner can close cases. When closed:

  • Court-house forum thread is deleted
  • If Guilty: Case posted to #case-logs with appeal button
  • If Not Guilty: Case posted to #closed-cases
  • Verdict posted to #appeals channel
  • Broadcast to all subscribed servers' #closed-cases channels

Privacy & Anonymity

The bot includes privacy features to protect reporters and hide sensitive information from public view.

Anonymous Reporting

When filing a report (via Discord or web), users can choose to remain anonymous:

  • Discord: In the report modal, type "yes" in the "Stay Anonymous?" field
  • Web: Check the "Submit Anonymously" checkbox on the form

Anonymous reporters are displayed as "Anonymous Reporter" in public embeds.

Hidden User IDs

User IDs (both reporter and defendant) are hidden from public view in all case embeds. This protects user privacy and prevents harassment.

🔒 Public View

Shows defendant name, reason, and reporter name (or "Anonymous"). No IDs visible.

🔐 Owner View

Bot owner receives a DM with confidential details including all user IDs.

Confidential Details (Owner Only)

When a case is filed, the bot owner receives a private DM containing:

  • Defendant ID
  • Reporter ID
  • Reporter Username
  • Whether the report was anonymous

If the DM fails, this information is posted to the #admin-alerts channel instead.

Cooldowns

  • Reports: 5 minutes between submissions
  • Contact: 10 minutes between submissions
  • Appeals: 30 minutes between submissions

Database Schema

The bot uses SQLite with the following main tables:

cases

case_id, case_number, reporter_id, reporter_username, defendant_type, defendant_id, defendant_name, reason, evidence, status, verdict, repeat_offender, forum_channel_id, closed_by, created_at, closed_at

votes

vote_id, case_id, voter_id, voter_username, vote, created_at

appeals

appeal_id, case_id, appellant_id, appellant_name, reason, status, reviewed_by, created_at, reviewed_at

guild_settings

guild_id, cases_channel_id, receive_guilty_only, created_at
