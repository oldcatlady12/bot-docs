🏗 Classes & Objects

Classes are blueprints for creating objects. They bundle data (attributes) and functions (methods) together.

Basic Class Structure

class_basic.py Python
class Player: """A class representing a sports player""" def __init__(self, name, team, position): # __init__ is the constructor - runs when creating an object self.name = name # instance attribute self.team = team self.position = position self.stats = {} # empty dict for stats def add_stat(self, stat_name, value): """Add a statistic for the player""" self.stats[stat_name] = value def __str__(self): # __str__ defines how to print the object return f"{self.name} ({self.position}) - {self.team}" # Create objects (instances) of the class lebron = Player("LeBron James", "Lakers", "SF") curry = Player("Stephen Curry", "Warriors", "PG") # Use methods lebron.add_stat("points", 28) print(lebron) # LeBron James (SF) - Lakers

⚡ Async Programming

Async (asynchronous) programming lets your bot do multiple things at once without waiting. This is crucial for Discord bots!

Why Async?

Without Async (Blocking)

Fetch NFL
→
Wait...
→
Fetch NBA
→
Wait...

With Async (Non-Blocking)

Fetch NFL
Fetch NBA
Fetch MLB
→ All at once!

Key Keywords

Keyword Meaning
async def Defines an asynchronous function (coroutine)
await Wait for an async operation to complete
asyncio Python's async library

🔔 Events & Tasks

Events let your bot respond to things that happen in Discord. Tasks run code on a schedule.

Common Events

events.py Python
# Bot is ready and connected @bot.event async def on_ready(): print(f"{bot.user} is online!") # Someone sent a message @bot.event async def on_message(message): # Ignore bot's own messages if message.author == bot.user: return if "hello" in message.content.lower(): await message.channel.send("Hey there!") # IMPORTANT: Process commands after custom handling await bot.process_commands(message) # Someone joined the server @bot.event async def on_member_join(member): channel = member.guild.system_channel if channel: await channel.send(f"Welcome, {member.mention}!")

⚡ JavaScript Async/Await

JavaScript uses Promises and async/await for asynchronous operations, similar to Python.

Promises

A Promise represents a value that might not be available yet.

promises.js JavaScript
// Creating a Promise const fetchData = new Promise((resolve, reject) => { // Simulate API call setTimeout(() => { const success = true; if (success) { resolve({ score: 100 }); } else { reject("Failed to fetch"); } }, 1000); }); // Using the Promise with .then() fetchData .then(data => console.log(data)) .catch(error => console.log(error));

Async/Await (Cleaner Syntax)

async_await.js JavaScript
// async function - can use await inside async function getStats() { try { // await pauses until Promise resolves const totalCases = await dbGet(db, 'SELECT COUNT(*) FROM cases'); const openCases = await dbGet(db, "SELECT COUNT(*) FROM cases WHERE status = 'OPEN'"); return { totalCases, openCases }; } catch (error) { console.error('Error:', error); } } // Arrow function version const getStats = async () => { const result = await dbGet(db, 'SELECT * FROM stats'); return result; };
