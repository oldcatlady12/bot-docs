Changelog Bot

Overview

The Changelog Bot receives update notifications from other bots and posts announcements to Discord. It uses a push-based system where bots send updates when changes happen, ensuring instant and reliable notifications.

How It Works

Push-Based Architecture

Instead of polling files, the changelog bot runs a webhook server. When any bot has an update, it sends a notification to the webhook, and the changelog bot immediately posts to Discord.

  1. Changelog bot starts a webhook server on port 5050
  2. When you update a bot, run !announce_update from that bot
  3. The bot sends the update to the changelog webhook
  4. Changelog bot posts a formatted embed to all configured channels
  5. Users with alert roles get notified instantly

Sending Updates from Other Bots

Each bot has an !announce_update command to send changelog updates:

Python Bots (Sports, Schedule, etc.)

!announce_update v1.2.0 - Added new live score tracking - Real-time updates every 30 seconds - Fixed timezone issues - Improved performance

JavaScript Bots (WorldGate)

/announce_update version:v1.2.0 changes:Added anonymous reporting feature

Programmatic Usage (Python)

from changelog_notifier import send_changelog_update await send_changelog_update( bot_key='sports', version='v1.2.0', changes='- Added feature X\n- Fixed bug Y' )

Programmatic Usage (JavaScript)

const { sendChangelogUpdate } = require('./changelog_notifier.js'); await sendChangelogUpdate({ botKey: 'worldgate', version: 'v1.2.0', changes: '- Added feature X\n- Fixed bug Y' });

Adding a New Bot

To register a new bot to send changelog updates:

  1. Add the bot to BOTS_CONFIG in bot.py:
    'newbot': { 'name': 'New Bot Name', 'emoji': '🌟', 'color': 0x3498db, # Hex color }
  2. Copy the changelog_notifier.py (or .js) helper to your bot's directory
  3. Add the !announce_update command to your bot
  4. Set the same CHANGELOG_WEBHOOK_SECRET in your bot's .env
