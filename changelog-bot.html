Changelog Bot

Monitors bot documentation files on GitHub Pages and automatically sends changelog notifications to configured Discord channels when updates are detected.

⚙ How It Works

1

Documentation Updated

A bot's HTML documentation page is modified and pushed to the GitHub Pages repository.

2

Trigger Detected

The bot polls for a trigger.json file every 3 seconds. When found, it reads the commit message and deletes the trigger.

3

Hash Comparison

MD5 hashes of all documentation files are computed and compared against stored hashes to detect which files changed.

4

Notifications Sent

Formatted embed messages with "About" buttons are sent to all configured changelog channels across servers.

✨ Features

📰 Change Detection

Uses MD5 hashing to detect when documentation files change. Compares current file hashes against stored snapshots to identify updates.

📣 Multi-Server Notifications

Sends changelog notifications to multiple Discord servers simultaneously. Each server can configure its own changelog channel.

💬 Commit Messages

Includes the git commit message in notifications so users know what changed. Supports both single and multi-bot update embeds.

🛠 About Buttons

Each notification includes interactive "About" buttons that show bot details including description, tech stack, and documentation link.

🌱 Seed Mode

Run with --seed flag to snapshot all current file hashes without sending notifications. Useful for initial setup.

📊 Multi-Bot Batching

When multiple documentation files change at once, they're batched into a single embed with inline fields and button rows.

🔧 Configuration

config.json

Key Type Description
token String Discord bot token
clientId String Discord application client ID

changelog_settings.json

Key Type Description
changelog_channels Object Map of guild IDs to channel IDs where changelogs are posted
file_hashes Object Stored MD5 hashes for each bot's documentation file
posted_hashes Object Hashes of messages already posted (deduplication)
alert_roles Object Roles to ping when changelogs are posted (per guild)

bots.json Entry Format

Key Type Description
name String Display name of the bot
filename String HTML documentation filename (e.g. sports-bot.html)
color String Hex color for embed (e.g. 0x2ECC71)
emoji String Emoji used in embed titles and buttons
description String Short description shown in "About" embeds
tech String Tech stack shown in embed footer

├── config.json # Bot token and client ID

├── changelog_settings.json # Channel config, hashes, state
├── package.json # Dependencies (discord.js v14)
└── utils/
    ├── changelogService.js # Core logic: detect changes, send notifications
    ├── embedBuilder.js # Build Discord embeds and buttons
    ├── hashService.js # MD5 hashing of documentation files
    └── logger.js # Console logging utility

🚀 Setup

1. Install

Run npm install in the changelog-bot directory to install discord.js v14.

2. Configure

Add your bot token and client ID to config.json. Add changelog channel IDs to changelog_settings.json.

3. Seed Hashes

Run npm run seed to snapshot current documentation hashes without sending notifications.

4. Start

Run npm start to begin polling. Create a trigger.json file to trigger change detection.
